Seed Data Overview

    
The schema seeds all required data automatically on first run. Here's what gets inserted:

    
-- 6 Brands: Nike, Adidas, Puma, H&M, Zara, Levi's
-- 8 Categories: tops, bottoms, dresses, sports_bras,
--               hoodies, accessories, outerwear, footwear
-- 48 Shop URLs: 6 brands × 8 categories
-- 36 Size Rows: 6 brands × 6 sizes (XS–XXL)
--   Each row has bust/waist/hips min-max ranges in cm

-- Size Logic Example (Nike):
-- XS: bust 79-84, waist 61-66, hips 86-91
-- S:  bust 84-89, waist 66-71, hips 91-96
-- M:  bust 89-94, waist 71-76, hips 96-101
-- L:  bust 94-100,waist 76-82, hips 101-107
-- XL: bust 100-107,waist 82-89,hips 107-114
-- XXL:bust 107-114,waist 89-96,hips 114-121

⚡ How To Run Weav AI

  
Full-stack: React Native (Expo) + FastAPI + MySQL. Three ways to run:


  

    
🌐 Option 1 — This HTML File (Instant, No Setup)

    
The LIVE APP tab above is fully functional with localStorage — no server needed. Perfect for demos and testing.

    

      
Demo Credentials

      
Emailtest@weavai.com

      
Passwordtest123

    

  


  

    
🐳 Option 2 — Docker (Recommended)

    

      
One command — MySQL + FastAPI + phpMyAdmin

      
cd weavai
docker-compose up --build

    

    
1
Backend API → http://localhost:8000

    
2
API Docs (Swagger) → http://localhost:8000/docs

    
3
Database GUI → http://localhost:8080

  


  

    
🐍 Option 3 — Manual Setup

    
Backend

    

      
cd weavai/backend
python -m venv venv
source venv/bin/activate    # Windows: venv\\Scripts\\activate
pip install -r requirements.txt
# Edit .env — set DB_PASSWORD
mysql -u root -p < schema.sql
uvicorn main:app --reload --port 8000

    

    
Frontend (React Native)

    

      
cd weavai/frontend
npm install
# Edit src/services/api.js — set BASE_URL to your machine IP
npx expo start
